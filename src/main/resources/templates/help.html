Setup Exam

Exam Paper:
The latest version of one of the existing exam papers.
For Module:
A module of which your are a module leader. All students in this module will have to take this exam
Date:
Date on which the exam is supposed to take place.
Start Time:
Start time of the exam. Exam will not start automatically. Privileged user will need to press start.
Location:
Venue in which the exam will take place.
Time allowed:
Amount of time the candidates will have to complete exam. 50% extra time will automatically be granted to candidates who require it.

Create Question

Language:
If a question is related to any programming language, this field should be populated.
Reference Name:
The name of the question that will be visible when browsing content. It won't be presented to candidates taking the exam.
Max Mark:
Maximum mark that can be awarded for answering this question correctly.
Proposed Time Scale:
Time scale in minutes which will be used to calculate proposed time scale of a test paper.
Type:
Type of question:
  • Code: Gives the candidate a text area with simple code highlighting support. Does not support auto marking.
  • Essay: Gives the candidate a text area to support long answers. Does not support auto marking.
  • Multiple Choice: Gives a candidate a choice of possible answers. Auto marking wizard will look at max mark and match an upper case letter followed by ')' in question text. You can use the update button to refresh the wizard if either has changed.
  • Drawing: Gives a candidate a simple computer graphics app. Does not support auto marking.
  • Expression: Asks the candidate to fill in up to 9 text boxes. Auto marking wizard match a digit from 1 to 9 surrounded by '[[' ']]'. Then you'll be able to add any number of possible answers for each of the blanks. The wizard check boxes are used to generate regex. White Space Collapsing: Multiple white spaces are treated as single. Alternative Punctuation: punctuation in the answer can be treated as any other punctuation mark or single white space. Case Insensitive: answer will be case insensitive. Alternatively you can write your own regex by pressing customise regex button.
Question Text:
The question text that will be displayed to the candidate. It uses a WYSWIG editor which supports styling. In order to enable code highlighting select a paragraph of text, press "Paragraph Format" and select code. Then press "Paragraph Style" and select code or sql.
Marking Guide:
The marking guide will be displayed to the marker to help them pick the correct mark. It uses a WYSWIG editor which supports styling. In order to enable code highlighting select a paragraph of text, press "Paragraph Format" and select code. Then press "Paragraph Style" and select code or sql.

Create Section

Reference Name:
The name of the question that will be visible when browsing content. It will also be presented to candidates taking the exam.
Number of Questions to Answer:
Sometimes candidates won't be required to answer all questions presented in a section. Pick a number equal to number of questions candidates are required to answer in this section.
Proposed Time to Complete:
Time scale in minutes which will be used to calculate proposed time scale of a test paper.
Instructions Text:
The section instructions text will be presented to candidates at the beginning of the section. It uses a WYSWIG editor which supports styling. In order to enable code highlighting select a paragraph of text, press "Paragraph Format" and select code. Then press "Paragraph Style" and select code or sql.
Section Questions:
Questions in this section. Questions can be added to a section once it has been created. Questions can be reordered by dragging their number up or down.

Create Paper

Reference Name:
The name of the paper that will be visible when browsing content. It will also be presented to candidates taking the exam.
Proposed Time to Complete:
Time scale in minutes which will be used to propose the time scale of the exam. The paper proposed time to complete can be overridden for each exam.
Instructions Text:
The section instructions text will be presented to candidates at the beginning of the section. It uses a WYSWIG editor which supports styling. In order to enable code highlighting select a paragraph of text, press "Paragraph Format" and select code. Then press "Paragraph Style" and select code or sql.
Section Questions:
Sections in this paper. Sections can be added to a paper once it has been created. Sections can be reordered by dragging their number up or down.
